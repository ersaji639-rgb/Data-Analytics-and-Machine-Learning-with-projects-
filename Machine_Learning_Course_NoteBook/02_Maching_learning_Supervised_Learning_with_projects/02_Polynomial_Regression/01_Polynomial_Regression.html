    Bias–Variance Trade-off¶

  • Bias: Error due to overly simplistic assumptions in the model.

    • High bias → Underfitting
    • The model misses relevant relations between input and output.
    • Example: Linear model used to fit nonlinear data.

  • Variance: Error due to too much complexity in the model.

    • High variance → Overfitting
    • The model captures noise as if it were a pattern.
    • Example: Deep decision trees memorizing training data.

  • Trade-off:

    • A model with too much bias performs poorly on training and test data.
    • A model with too much variance performs well on training data, but poorly on unseen data.
    • The goal is to find a sweet spot where both bias and variance are minimized enough to generalize well.

Summary¶

Model Behavior Bias Variance Training Error Test Error
Underfitting High Low High High
Overfitting Low High Low High
Good Generalization Balanced Balanced Low Low (ideally)

🔁 Bias–Variance Trade-off¶

  • Bias: Error due to overly simplistic assumptions in the model.

    • High bias → Underfitting
    • The model misses relevant relations between input and output.
    • 🧠 Example: Trying to fit a straight line to curved data (e.g., using Linear Regression for a quadratic relationship).

  • Variance: Error due to too much complexity in the model.

    • High variance → Overfitting
    • The model captures noise as if it were a pattern.
    • 🧠 Example: Using a deep decision tree on a small dataset, leading to perfect training accuracy but poor generalization.

  • Trade-off:

    • A model with too much bias performs poorly on training and test data.
    • A model with too much variance performs well on training data, but poorly on unseen data.
    • ✅ Goal: Find a sweet spot where both bias and variance are balanced for good generalization.

✅ Summary¶

Model Behavior Bias Variance Training Error Test Error Example
Underfitting High Low High High Linear Regression on nonlinear data
Overfitting Low High Low High Deep tree on small/noisy dataset
Good Generalization Balanced Balanced Low Low (ideal) Regularized model / tuned model

🎯 Is Bias–Variance Trade-off a Spectrum?¶

Yes — the bias–variance trade-off is best understood as a spectrum, not a binary concept.

🔵⚖️🔴 Bias–Variance as a Sliding Scale¶

  • High Bias / Low Variance → Underfitting

    • Model is too simple to capture the true pattern in the data.
    • Example: Fitting a straight line to curved data.

  • Low Bias / High Variance → Overfitting

    • Model is too complex and learns noise as if it were a pattern.
    • Example: Deep neural network trained on small noisy data.

  • ✅ Goal: Find a sweet spot where bias and variance are balanced, leading to good generalization on unseen data.

📉 Bias–Variance Curve (Conceptual)¶

A common plot shows:

  • Model complexity on the x-axis
  • Prediction error on the y-axis

As model complexity increases:

  • Bias decreases
  • Variance increases
  • Total error first drops (better), then rises (worse)

Adjusting the parameter¶

from sklearn.preprocessing import PolynomialFeatures
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error
import numpy as np

# Lists to store errors
training_rmse_error = []
test_rmse_error = []

for d in range(1, 10):
    # Create a polynomial converter for each degree 
    polynomial_converter = PolynomialFeatures(degree=d, include_bias=False)
    polynomial_features = polynomial_converter.fit_transform(X)

    # Split into train and test set 
    X_train, X_test, y_train, y_test = train_test_split(
        polynomial_features, y, test_size=0.3, random_state=101
    )

    # Train the model 
    poly_model = LinearRegression()
    poly_model.fit(X_train, y_train)

    # Predictions on training and testing data
    train_predictions = poly_model.predict(X_train)
    test_predictions = poly_model.predict(X_test)

    # Calculate RMSE
    train_RMSE = np.sqrt(mean_squared_error(y_train, train_predictions))
    test_RMSE = np.sqrt(mean_squared_error(y_test, test_predictions))

    # Append the errors
    training_rmse_error.append(train_RMSE)
    test_rmse_error.append(test_RMSE)

🧠 Model Evaluation: Why Not Just One Train-Test Split?¶

  • Using a fixed random state and fixed test size gives only one snapshot of model performance.
  • This can lead to biased or high-variance estimates of error.
  • So how do we know what random state or test percentage is “good”?

✅ Better Approach: Cross-Validation¶

Cross-validation helps evaluate a model's generalization performance by splitting the data multiple times in a structured way.

⭐ Techniques¶

1. K-Fold Cross-Validation¶

  • The dataset is split into k equal-sized folds.
  • Each fold is used once as a test set, the remaining k-1 folds are used as training set.
  • This process is repeated k times.
  • The average of the test errors gives a robust estimate of model performance.
from sklearn.model_selection import cross_val_score
from sklearn.linear_model import LinearRegression
import numpy as np

model = LinearRegression()
scores = cross_val_score(model, X, y, cv=5, scoring='neg_root_mean_squared_error')
print("Average RMSE:", -np.mean(scores))

🔧 What is a Hyperparameter?¶

A hyperparameter is a configuration that is set before the learning process begins and not learned from the data. These values control the training process and influence the performance of a machine learning model.

📌 Examples of Hyperparameters:¶

Model Type Hyperparameters (Examples)
Linear Regression Regularization strength (alpha in Ridge/Lasso)
Decision Trees max_depth, min_samples_split
K-Nearest Neighbors n_neighbors, distance metric
Neural Networks Learning rate, number of layers, batch size
General (any model) Number of training epochs, optimizer type

🧠 Hyperparameters vs Parameters:¶

Aspect Hyperparameters Parameters
When Set Before training Learned during training
Examples Learning rate, tree depth Weights in linear regression or neural nets
Determined by Developer/optimizer (via tuning) Training algorithm (via data)

⚙️ How to Choose Hyperparameters?¶

  • Manually (trial and error)
  • Grid Search
  • Random Search
  • Cross-validation: Evaluate combinations robustly
  • Bayesian Optimization, Genetic Algorithms, etc.
